Redesigning the screens

Homescreen

Redundant banner slides on the home screen with misleading VoiceOver.


Redesign
  • Decluttered the page and made it intuitive.
  • Modified the orientation of the tabs to be more spaced out for low-vision users and those with hand dexterity.
  • Added color groups for quick identification of similar set of information.
Purchasing Train Ticket - Process
Previous
  • Multiple UI icons and images were not labelled or had Alt text, with the lack of function explanation.
  • Final ticket details were not announced.
  • State of the icons was not highlighted - Favorite (star), reduce (-) and add (+) icon.
  • 'Loading screen' animation gesture was not announced on Talkback, thus confusing the user about the page and their phone.
Redesign
  • Introducing an ‘Accessibility Filter’ for station selection.
  • Multiple choices for the origin station selection.
  • Color identification for similar set of information.
  • An elastic search option with an inbuilt feature to display the nearest station to the user.
  • Users can search a station by its name or name of the area.
  • Adjusted the spacing between elements for better navigation and click ability.
  • Station names are made into Title case than an entire Uppercase for readability.
Purchasing Train Ticket - Final Screen
Redesign
  • Ensured the colors on dark and light UI are as per the WCAG guidelines.
  • Simplified the layout of ticketing process with iconography and consistent elements.
  • Revised the 'Alt text' of + / - and rest of the icons to announce the number of selected tickets or relevant information of the tags respectively.
Departure Vision (Status)
Previous
  • Overall flow was hard to follow.

Redesign - Departure “Status”
  • Renamed as Departure Status from Departure Vision to ensure the correct meaning of the tab.
  • Can search the station for desired path/route in the same app
  • Similar flow to ‘Purchasing ticket.’
